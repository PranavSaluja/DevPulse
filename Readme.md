## What is DevPulse?

DevPulse is a **hosted GitHub App** that automatically analyzes pull requests and posts AI-generated review insights directly on your PR — before a human reviewer even opens it.

When a pull request is opened, DevPulse:

1. Receives the pull request event from GitHub via webhook
2. Publishes the event to a **Kafka topic** for asynchronous processing
3. Fetches the PR diff using the GitHub REST API
4. Sends the diff to **Google Gemini** for AI analysis
5. Posts a structured review comment directly on the pull request

> **The goal:** Instant, automated, and consistent PR feedback — integrated seamlessly into your existing workflow.

## The Problem

Manual PR reviews are slow, inconsistent, and cognitively expensive.

| Challenge | Impact |
|---|---|
| Large diffs are hard to evaluate | Risky changes get missed |
| Review quality varies by reviewer | Inconsistent code standards |
| PRs wait for busy reviewers | Development velocity blocked |
| No automated contextual insights | Issues merge into production |

## Architecture

```
Developer opens Pull Request
          ↓
GitHub sends webhook event
          ↓
┌─────────────────────────────┐
│   Webhook Service           │
│   Java + Spring Boot        │  ← Hosted on Railway
│   POST /github/webhook      │
└────────────┬────────────────┘
             ↓
┌─────────────────────────────┐
│   Kafka Topic: pr.created   │  ← Upstash (Managed Kafka)
└────────────┬────────────────┘
             ↓
┌─────────────────────────────┐
│   AI Review Service         │
│   Python + FastAPI          │  ← Hosted on Render
│   Consumes Kafka events     │
└────────────┬────────────────┘
             ↓
     Google Gemini API
             ↓
  PR Comment posted on GitHub
```

### Services

| Service | Language / Framework | Responsibility |
|---|---|---|
| `webhook-service` | Java 21 + Spring Boot 3 | Receive GitHub events, verify HMAC signature, publish to Kafka |
| `ai-review-service` | Python 3.11 + FastAPI | Consume Kafka events, fetch PR diff, call Gemini, post comment |
| Kafka Broker | Upstash (Managed) | Async event bus decoupling the two services |

## Local Development

### Prerequisites

- Docker + Docker Compose
- Java 21
- Python 3.11+
- A GitHub App (see setup below)
- [smee.io](https://smee.io) for local webhook tunneling

### 1. Clone the repository

```bash
git clone https://github.com/YOUR_USERNAME/devpulse.git
cd devpulse
```

### 2. Configure environment variables

Create a `.env` file in the root:

```env
# GitHub App
GITHUB_APP_ID=your_app_id
GITHUB_PRIVATE_KEY=your_private_key_pem
GITHUB_WEBHOOK_SECRET=your_webhook_secret

# Kafka (local)
KAFKA_BOOTSTRAP_SERVERS=localhost:9092

# Gemini
GEMINI_API_KEY=your_gemini_api_key
```

### 3. Start services

```bash
docker-compose up
```

This starts Zookeeper, Kafka, the Webhook Service, and the AI Review Service together.

### 4. Tunnel webhooks locally

```bash
smee --url https://smee.io/YOUR_CHANNEL --path /github/webhook --port 8080
```

## GitHub App Setup

1. Go to **GitHub → Settings → Developer Settings → GitHub Apps → New GitHub App**
2. Set the webhook URL to your public endpoint (or smee.io URL for local dev)
3. Configure permissions:

| Permission | Access |
|---|---|
| Pull Requests | Read |
| Repository Contents | Read |
| Issues | Write |

4. Generate a private key and download it
5. Install the app on your target repository

## Deployment

DevPulse is deployed using a fully free hosting stack.

### Kafka — Upstash

1. Create a free account at [console.upstash.com](https://console.upstash.com)
2. Create a Kafka cluster named `devpulse`
3. Create topic `pr.created`
4. Copy your `BOOTSTRAP_SERVERS`, `USERNAME`, and `PASSWORD`

### Webhook Service — Railway

1. Connect your GitHub repo at [railway.app](https://railway.app)
2. Point to `/webhook-service`
3. Add environment variables in the Railway dashboard
4. Copy the generated public URL (e.g. `https://devpulse-webhook.up.railway.app`)

### AI Review Service — Render

1. Connect your GitHub repo at [render.com](https://render.com)
2. Point to `/ai-review-service`
3. Set build command: `pip install -r requirements.txt`
4. Set start command: `uvicorn main:app --host 0.0.0.0 --port 8000`
5. Add environment variables

### Update GitHub App Webhook URL

Set the webhook URL in your GitHub App settings to your Railway public endpoint:

```
https://devpulse-webhook.up.railway.app/github/webhook
```
